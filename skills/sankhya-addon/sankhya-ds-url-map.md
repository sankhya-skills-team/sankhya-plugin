## 9. Runtime de tela DS em ADDON — gotchas validados em campo (addon-exemplo)

Tela DS (`snk-crud`/`snk-application`) registrada no menu como `<ui url="/$ctx/labsApps/<Tela>/build/index.html">` **renderiza** (pipeline Node OK) mas quebra em runtime porque o addon roda em contexto próprio (`/addon-exemplo/...`) sem o runtime do shell e sem a sessão do BFF. Correções (no `index.html`/`App.tsx`, antes de montar `SnkApplication`):

1. **`window.APPLICATION_NAME = "<Tela>"`** — senão o path de mensagens vira `labsApps/undefined/...` (404).
2. **Bridge `utxt`** do `window.parent`/`window.top` — `utxt` é global do `sf.js` do host (decode da lista de autorização); sem ela: `ReferenceError: utxt is not defined` em cascata. `window.utxt = window.parent.utxt`.
3. **Warm-up do BFF (bloqueio principal):** GraphQL vai pra `/${getModuleName()}/graphql` (default `mgefin-bff`) e cold dá 500 `Sessão MGE não iniciada`. Antes de montar o `SnkApplication`, gateie o render e faça `await fetch("/mgefin-bff/DynaformLauncher.xhtml5?mgeSession="+token, {credentials:"include"})` — o launcher do BFF registra a sessão e seta o `JSESSIONID` do contexto; o graphql seguinte vira 200. Equivale a abrir manualmente uma tela do módulo antes.
4. **Deploy labsApps:** além de `build/`, o workspace espera `labsApps/<Tela>/package.json` + `build/module_structure.json` (`npm ls --depth=0 --json`). Gere via task Gradle própria (NÃO redefina `compileDS` — é nativa do plugin `addonstudio`; crie agregadora e amarre a `buildWar`/`deployAddon`).
5. **Vite:** `index.html` na RAIZ (não em `public/` — isso é CRA; a doc `getting-started/configure` está desatualizada).
6. **Diag:** `curl /{bff}/graphql?mgeSession=TOKEN` sem cookie=500, com `-H "Cookie: JSESSIONID=TOKEN"`=200 → confirma que é registro de sessão no BFF.

> Tudo isso é **workaround** do gap "labsApps de addon × BFF de core". `ez-*` puro (sem `snk-application`/graphql) normalmente não precisa de 2/3.
